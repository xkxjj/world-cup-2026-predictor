投注倍率只代表模型给出的公允赔率。若实际平台赔率高于模型公允赔率，才可能存在模型意义上的“价格更高”；这仍不等于投注建议。

比分预测逻辑：当前比分基于国家队 Elo、2026 世界杯实际分组/赛程、已赛比分开关和自动同步赛果。新版不再把“全场平均总进球”硬拆给两队，而是用 Elo 派生攻击强度和防守强度：球队A期望进球 = 基准进球 * 比赛节奏 * A攻击强度 / B防守强度。

常量因素复盘：前 20 场真实赛果显示，原模型的基准总进球偏低、Elo 强弱差过度自信、主办国加成偏保守，所以默认值已调整为更开放的 2.75 球、更平滑的 Elo 缩放和更高的东道主加成。

变量因素复盘：已赛样本里双方进球、冷平和 4 球以上比赛明显不少，因此模型加入“赛果气候”和“比分方差”：赛果气候会随真实比赛滚动修正总进球、平局、双方进球；比分方差会按强弱差和东道主比赛提高尾部分布。

低比分修正：具体比分仍用泊松分布逐个计算，但对 0-0、1-0、0-1、1-1 加入 Dixon-Coles 低比分相关修正。负向相关参数会提高 0-0 和 1-1 的相对权重，同时压低部分过度集中的一球小胜，减少独立泊松对低比分结构的偏差。

比分排序：对 0-0 到 10-10 的所有比分计算修正后概率，再按命中概率从高到低排序。页面展示的比分选择就是这个排序，不额外插入某个大比分。

数据校准：若你在“数据校准”里导入十进制盘口，模型会把胜平负和大小球盘口转成隐含概率，再对每个比分做轻度再加权。盘口不是硬覆盖模型，只是把市场共识作为一层保守校准；没有盘口的比赛不会虚构数据。

状态校准：纳入已赛结果后，后续比赛会根据球队目前进球、失球和积分表现微调攻击、防守和比赛节奏。验真时不会偷看未来赛果，验证某一场只使用那场之前已经发生的比赛。

赛事情境：小组第三轮会根据前两轮积分加入战意修正：已基本锁定出线的球队降低强度，必须抢分的球队提高强度；如果双方平局都较安全，会降低节奏并提高低比分平局权重。

模拟流程：每轮模拟都会跑完 72 场小组赛、官方 32 强第三名分配、16 强到决赛。每条模拟路径会根据比赛结果微调动态锦标赛 Elo，让爆冷或连胜影响后续比赛。

淘汰赛与小组排序：淘汰赛 90 分钟打平后，点球/加时胜率会向 50-50 压缩，避免强队在点球大战中被 Elo 过度偏袒。小组排序补入同分球队之间的胜负关系，再用积分、净胜球、进球数、胜场和 Elo 兜底。

来源：赛制与赛程参考 国际足联赛程页、国际足联赛事规程；Elo 参考 国家队 Elo 评分表；已赛比分会优先读取同站 data/games.json，这个文件可由 GitHub Actions 定时从 WorldCup26 公开赛果接口更新；数据不可用时使用本地内置赛果。
